버전: release-0.5

리소스 모니터링

✓ MIG가 설정된 GPU는 GPU 사용률을 조회할 수 없습니다.

1. 기본 제공 뷰​

  1. Node Performance 뷰

    시스템의 종합적인 상태 파악과 성능을 관리할 수 있는 뷰입니다.

    • 모니터링 리소스: 노드의 성능, 작업 부하, 메모리 및 디스크 상태, 네트워크 트래픽 등

  2. GPU State 뷰

    GPU의 성능, 안정성, 온열 상태, 전력 효율성 등을 모니터링할 수 있는 뷰입니다.

  3. Slurm Scheduler Information 뷰

    스케줄러 대기 시간 파악 및 성능을 모니터링 할 수 있는 뷰입니다.

2. 노드 선택​

좌측 상단에서 모니터링할 노드를 선택할 수 있습니다.

3. 사용자 정의 뷰 추가​

모니터링 ▷ 리소스 알람 ▷ 상단 + 버튼 클릭 ▶️ 모니터링 할 유형 선택 ▶️ 추가 버튼 클릭

사용자는 자신의 용도와 목적에 맞는 뷰를 추가 가능

사용자 정의 뷰에서는 리소스 선택 가능하며, 차트의 크기와 배치도 조정 가능

4. 차트 추가​

모니터링하고자 하는 리소스 차트를 추가합니다.

  1. 뷰에 차트가 없는 경우

    모니터링 ▷ 리소스 모니터링 ▷ 좌측 + 버튼 클릭 ▶️ 리소스 선택 (다중 선택 가능) ▶️ 추가 버튼 클릭

  2. 뷰에 차트가 있는 경우

    모니터링 ▷ 리소스 모니터링 ▷ 우측 상단 + 버튼 클릭 ▶️ 리소스 선택 (다중 선택 가능) ▶️ 추가 버튼 클릭

5. 보조 차트 추가​

모니터링 ▷ 리소스 모니터링 ▷ 원하는 차트 선택 ▶️ 설정 아이콘 버튼 클릭

  1. 추가하고 싶은 보조 차트 체크 (다중 선택 가능) ▶️ 저장 버튼 클릭

  2. 보조차트 추가된 모습 확인

6. 차트 판넬 옵션 변경​

모니터링 ▷ 리소스 모니터링 ▷ 원하는 차트 선택 ▶️ 설정 아이콘 버튼 클릭

변경 가능한 옵션 항목​

  1. 제목 변경

  2. 실시간 모니터링 ON / OFF

    • ON : 실시간으로 데이터를 모니터링
    • OFF: 설정한 기간의 히스토리 데이터 모니터링

  3. 업데이트 주기

    데이터를 갱신하는 주기를 입력합니다. (단위: 초)

  4. 히스토리 조회 기간

    조회하고 싶은 히스토리 데이터의 기간을 입력합니다.

  5. 메인차트 타입

    어떤 차트를 메인으로 사용할 것인지 선택합니다.

    • 라인 차트
    • 바 차트
    • 텍스트 차트
    • 게이지 차트
    • 히트맵

  1. 데이터 모드

    • 원시 : 원시 데이터는 변형되지 않은 형태로 존재하며, 추가적인 처리나 가공없이 사용될 수 있습니다.
    • 보간 : 누락된 데이터를 0으로 채워줘 누락된 데이터 사이를 보간함으로 써 부드러운 곡선을 생성합니다.

  2. 보조 차트 추가

    메인차트와 함께 출력할 보조 차트를 선택합니다.

    • 시간별 차트
    • 요일별 차트
    • 월별 차트
    • 테이블 차트

  3. 범례 선택

    차트에 출력할 범례를 선택합니다.

    • 라인 차트, 바 차트, 테이블 차트, 히트맵
      • 다중 선택 가능
    • 텍스트 차트, 게이지 차트
      • 한 개만 선택 가능

8. 차트 판넬 위치 변경​

  1. 차트 판넬의 헤더를 클릭
  2. 원하는 위치에 drag & drop

9. 차트 판넬 크기 변경​

  1. 차트 판넬의 우측 ⌟ 버튼 클릭
  2. drag & drop으로 크기 변경

10. 메인 및 보조 차트 위치 변경​

  1. 메인 혹은 보조 차트의 우측 ✣ 버튼 클릭
  2. 차트 판넬 내에서 drag & drop으로 위치 변경

11. 메인 및 보조 차트 크기 변경​

  1. 메인 혹은 보조 차트의 우측 ⌟ 버튼 클릭
  2. drag & drop으로 크기 변경
