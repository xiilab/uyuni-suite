버전: release-0.5

kubernetes

1. 노드 추가​

클러스터 ▷ 좌측 상단 쿠버네티스 탭 버튼 클릭 ▶️ 우측 상단 + 노드 버튼 클릭 ▶️ 추가 버튼 클릭

현재 쿠버네티스에 추가 되어 있는 모든 노드를 우유니에 추가합니다.

2. 노드 조회​

우유니에 추가된 노드를 조회할 수 있습니다.

  1. 노드 우선 순위

  2. 노드 IP, 노드명

  3. MIG 태그

    • MIG 기능이 설정되어 있는 경우에만 출력됩니다.

  4. 해당 노드를 사용할 수 있는 그룹

  5. 노드의 리소스 평균 값

    • GPU, GPU Memory, GPU 온도, CPU, Memory, Disk

3. MIG 설정하기​

✓ MIG 설정 기능은 하단 NVIDA GPU 모델만 지원됩니다.

  • A100 40G
  • A100 80G
  • A30
  • H100

  1. MIG 활성화 하기

    클러스터 ▷ 좌측 상단 쿠버네티스 탭 버튼 클릭 ▷ 원하는 노드 선택 ▶️ 우측 MIG 버튼 클릭 ▶️ 좌측 profile 조합 방법 셀렉트 박스 클릭 ▶️ single 옵션 선택 ▶️ profile 셀렉트 박스 클릭 ▶️ 원하는 profile 클릭 ▶️ 우측 하단 확인 버튼 클릭

  2. MIG 비활성화 하기

    클러스터 ▷ 좌측 상단 쿠버네티스 탭 버튼 클릭 ▷ 원하는 노드 선택 ▶️ 우측 MIG 버튼 클릭 ▶️ 좌측 profile 조합 방법 셀렉트 박스 클릭 ▶️ MIG 비활성화 옵션 선택 ▶️ 우측 하단 확인 버튼 클릭

4. 노드 우선 순위 설정​

프로젝트 생성 시 설정한 노드 우선순위 정보를 바탕으로 GPU를 할당해줍니다.

클러스터 ▷ 좌측 상단 쿠버네티스 탭 버튼 클릭 ▶️ 우측 상단 노드 우선순위 버튼 클릭 ▶️ drag&drop 혹은 우측 화살표 버튼 클릭으로 순위 변경 ▶️ 변경 버튼 클릭

5. 노드별 그룹 설정​

클러스터 ▷ 좌측 상단 쿠버네티스 탭 버튼 클릭 ▶️ 우측 상단 노드별 그룹 설정 버튼 클릭 ▶️ 변경 버튼 클릭

✓ 그룹 설정에 어느 그룹도 설정 되어있지 않으면 모든 그룹이 사용 가능합니다.

  • 각 노드에 접근 가능한 그룹을 제한할 수 있습니다.
  • 설정된 그룹 외 다른 그룹은 해당 노드의 자원(GPU, Memory, CPU, Disk 등)을 사용할 수 없습니다.
